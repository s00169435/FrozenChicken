30th June 2021

A showcase of how the player movement system works. It's triggered by a mouse click on an onoccupied tile. Using the node positions on the selected tile and the BreathFirstSearch algorithm, the system will look for the shortest path to that selected tile. I'm gonna later adjust it so that it won't go on tiles that are marked as "unavailable" and maybe add a little icon above the selected node, so that the player will always know where they are going. The player's speed is also affected by obstacles in the way, so they will be slowed down if they're not careful about their movement.
